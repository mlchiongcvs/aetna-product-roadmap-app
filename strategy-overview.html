How the 4 Platform Layers Transform Each Experience
Each platform is shifting from legacy, batch-driven operations to modern, real-time, AI-powered capabilities — and every user group benefits

Integration Platform

Today
Point-to-point batch file exchanges (834s, encounters, vendor feeds) stitched together with legacy Siteminder SSO. Data moves in overnight batches — eligibility changes lag 24–48 hours. Each new state requires custom integration builds.
By 2028
Real-time event-driven architecture: Ping Identity SSO, Aetna Transactional Cache (ATC) for streaming, Azure event-driven pipelines, GPS as unified hub. Eligibility, claims status, and benefit changes propagate in near real-time.
Impact by User Group
Member
Digital ID card day one; real-time benefit confirmation without calling
Caregiver
Instant visibility into member status changes — no next-day lag
Provider
Real-time claims status and eligibility via Availity — not phone calls
Case Manager
Live member data in OneCM — no stale records from yesterday's batch
Clinician
MedCompass gets UM data in real-time; auth decisions flow instantly to providers

Transactional Platform Claims · Constituent Service · Clinical · Network & Provider

By 2028
QNXT upgraded and consolidated. ATLAS replaces EOL claims workflow (Jan 2027). OneCM (Salesforce Health Cloud) unifies care management. PRMS streamlines credentialing. AI claims exception handling targets 90%+ auto-adjudication.

Core Admin Platform Enrollment · Benefits · Plan · Health Plan Admin
